Descripción del flujo de trabajo

GitHub flow es un flujo de trabajo ligero que soporta equipos y proyectos que se despliegan regularmente. Ya sea para estudiantes o para profesionales, GitHub flow es una excelente opción para colaborar en proyectos de cualquier tamaño.

Características:

  • Crear una rama para cada tarea.
  • Realizar cambios en la rama.
  • Realizar un pull request.
  • Revisar el código.
  • Realizar un merge.

Comandos de Git que se usan:

Comando Funcionalidad
git clone Clona un repositorio remoto en la carpeta actual.
git checkout -b Crea una rama y se cambia a ella.
git add Añade archivos al área de preparación.
git commit Guarda los cambios en el repositorio.
git push Sube los cambios al repositorio remoto.
